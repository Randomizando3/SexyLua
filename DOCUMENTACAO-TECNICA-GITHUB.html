1. Visão geral

SexyLua é uma plataforma premium para criadores, assinantes e operação administrativa. O produto combina catálogo público, assinaturas, chats privados, microconteúdos, packs, live streaming, carteira em LuaCoins e integração PIX.

Stack principal

  • PHP 8.3+
  • Templates HTML renderizados no servidor
  • JavaScript pontual para interações específicas
  • Persistência em JSON por padrão
  • SyncPay para PIX
  • MediaMTX para lives

Papel do repositório

  • Guardar o código do produto
  • Servir como base para manutenção contínua
  • Permitir publicação segura sem segredos

4. Configuração e bootstrap

O bootstrap parte de public/index.php e carrega bootstrap.php. O ambiente usa .env com base em .env.example. A estrutura suporta tanto árvore simples quanto layout com private/app.

Variáveis importantes

  • SEXYLUA_STORAGE_DRIVER: json ou postgresql
  • SEXYLUA_BASE_URL: URL base da aplicação
  • SEXYLUA_LIVE_DRIVER: mediamtx
  • SEXYLUA_MEDIAMTX_API_URL: API do MediaMTX
  • SEXYLUA_MEDIAMTX_RTMP_URL: ingest RTMP
  • SEXYLUA_MEDIAMTX_HLS_URL: playback HLS

5. Pagamentos e wallet

  • Recarga PIX via SyncPay
  • Crédito só após aprovação/retorno válido
  • Wallet interna em LuaCoins
  • Admin com revisão de recargas, saques, ajustes e exportação XLSX
  • Webhook padrão: /webhook/syncpay

6. Lives

O fluxo de live usa MediaMTX para ingest RTMP e entrega HLS. O replay automático permanece desligado por padrão para reduzir disco e processamento.

  • Preset recomendado: 854x480
  • Vídeo: 800 kbps
  • FPS: 30
  • GOP: 2s
  • Áudio AAC: 96 kbps

7. Deploy genérico

  1. Preparar a estrutura pública e a árvore da aplicação.
  2. Publicar o código.
  3. Configurar as variáveis de ambiente.
  4. Ajustar as integrações pelo painel admin.
  5. Garantir permissões de escrita em uploads e storage.
  6. Validar home, login, chat, wallet, conteúdo e lives.

8. Regras para GitHub

  • Não versionar .env real.
  • Não versionar cookies, back-ups, arquivos sensíveis ou dados privados.
  • Não publicar storage real de produção.
  • Manter a documentação genérica livre de segredos.

9. Checklist técnico mínimo

Antes de qualquer entrega: testar cadastro, login, perfil público, wallet, SyncPay, chat, compra avulsa, live, darkroom e financeiro do admin.
